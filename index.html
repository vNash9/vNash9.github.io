MSc Thesis — Supervised by Dr. Julian Kunkel, GWDG
A Multi-Dimensional Evaluation Framework for LLM Inference Optimization in Production AI-as-a-Service Environments
S
Situation

Deploying LLMs in production means choosing between plethora of parameters, FP16 vs INT4 quantization, GPTQ vs AWQ, vLLM vs TensorRT-LLM. Wrong choice = wasted GPU budget & latency SLO violations. Currently: trial and error. Existing benchmarks (HELM, lm-eval) test model quality, not production inference.

T
Task

Design a systematic framework that evaluates LLM inference across multiple dimensions simultaneously and automatically recommends optimal deployment configurations based on user-defined constraints.

A
Action

Built end-to-end benchmarking harness on vLLM testing across concurrency levels (1→2000+ users), multiple quantization strategies, GPU telemetry at 1Hz. Measuring TTFT, P50/P95/P99 latency, throughput, cost-per-token. Developing automated recommendation engine with constraint-based filtering and Pareto-optimal configuration selection.

R
Result

Framework can be integrated into GWDG's SAIA Platform. Data-driven deployment decisions replacing guesswork. Expected 1.5–3× throughput improvements from optimal configuration selection. Reduces expertise barrier — Just plug in input constraints & get recommendations.
